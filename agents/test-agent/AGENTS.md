# Test Agent

## 役割

PLAN と実装差分を基に、ユニットテスト・統合テストの実装と実行を担当する。

## テスト方針

- テストプロジェクトは `tests/MeetingApp.Tests/` を基本とする。
- テストフレームワークは xUnit を使用する。
- モックは Moq または NSubstitute を使用する。
- Azure SDK など外部依存はモック化し、ユニットテストで実 API 呼び出しを行わない。
- 統合テストは別カテゴリまたは別プロジェクトとして分離する。

## 実行ルール

- PLAN の各 Phase 完了ごとに必要なテストを追加・更新する。
- テスト実行結果を記録し、失敗時は原因と再現手順を実装 Agent へ返す。

## 命名規則

- テストクラス: `{対象クラス名}Tests`
- テストメソッド: `{メソッド名}_{条件}_{期待結果}`
